## Why This Lab Exists

In the world of Systems Administration, theory only gets you so far. This portfolio is a living documentation of my journey through **Infrastructure as Code**, **Network Security**, and **Server Orchestration**.

The goal of this blog is threefold:

1. **Accountability:** Documenting my progress through platforms like _OverTheWire_ and _SadServers_.
2. **Knowledge Base:** Creating a "Searchable Brain" for complex fixes I've implemented, like point-in-time database recovery.
3. **Visibility:** Providing evidence of my technical hands-on skills for future employers.

---

## The Roadmap

My current home lab roadmap is divided into three major phases:

### Phase 1: The Foundation (Current)

- Master the Linux CLI and basic shell scripting.
- Deploy a hardened LAMP stack with automated backups.
- Secure SSH access using RSA Key Pairs (Standardizing security).

### Phase 2: Virtualization & Networking

- Migration to a dedicated Type-1 Hypervisor (Proxmox VE).
- Implementation of VLANs to isolate "Production" and "Test" environments.
- Setting up centralized logging (ELK Stack or Graylog).

### Phase 3: Modern Operations (Cloud & DevOps)

- Containerizing applications using **Docker** and **Portainer**.
- Automating server configuration with **Ansible**.
- Implementing CI/CD pipelines for static site deployment (like this Jekyll site!).
